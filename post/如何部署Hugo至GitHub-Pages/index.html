如何部署Hugo至GitHub Pages

2022年4月12日
| 阅读

先决条件

  • 你已在本地部署 Hugo
  • 你已拥有 GitHub 账号并下载了 Github Desktop

生成最终网页

hugo --theme=xxxxx --baseUrl="http://xxxxx.github.io/"

BaseUrl后填写你的网址，也就是GitHub（xxxxx.GitHub.io）或你要绑定的域名 theme后填你使用的主题文件夹名

千万不要忘记在网址最后加上/，否则你会被debug折磨致死

配置 Github Desktop

  1. 双击 Github Desktop，然后登录你的账号
  2. 点击左上角 file，点击 new repository
  3. name填你的用户名.GitHub.io，当然，如果你要绑定自己的域名命名就无所谓了
  4. Local path随便选，GitHub 会初始化这个目录为仓库
  5. 点击 Create repository
  6. 进入你选择的目录（local path），将hugo生成的public文件夹内所有内容移至local path
  7. 打开 Github Desktop，填写 summary（随便填）并点击 commit to master（或main）

配置 GitHub Pages

  1. 进入GitHub网页端，进入仓库，点击 settings，往下翻到 Pages
  2. 点击check什么什么（记不住了）
  3. 找到 Souece Branch 选择 Master 或 Main 后面的选/root
  4. all done Github Pages页面中有这样一段话（Your site is published at ……..）
  5. 点击网址，查看最终效果

后记

如果你的css样式没有生效，可以尝试将 config.toml 中的 languageCode 改为 zh-CN
